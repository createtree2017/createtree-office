# 기능 계획서: UI/UX 고도화 및 배포 준비 (Phase 4)

## 1. 개요
*   **작성일**: 2026-03-03
*   **기능명**: UI/UX 폴리싱 및 배포 (Deployment)
*   **목적**: 개발된 사내 업무 시스템(createTree Office)의 사용자 경험을 극대화하고, 실제 운영 환경에 배포하기 위한 최종 준비를 진행합니다.

## 2. 주요 작업 내용 (Plan)

### A. UI/UX 폴리싱 (프론트엔드)
1.  **업무 관리 (Tasks) 보드 UX 개선**
    *   현재 상태별로 분류된 칼럼 간에 카드를 마우스로 끌어서 이동할 수 있는 **드래그 앤 드롭 (Drag and Drop)** 기능 구현. (`@hello-pangea/dnd` 등 라이브러리 검토)
    *   상태 변경 시 즉각적인 낙관적 업데이트(Optimistic Update) 적용.
2.  **전역 로딩 상태 처리**
    *   데이터를 불러오는 동안 사용자가 대기 상태임을 인지할 수 있도록 스켈레톤(Skeleton) UI 또는 글로벌 로딩 스피너 적용.
3.  **반응형(Responsive) 레이아웃 다듬기**
    *   모바일 환경에서도 대시보드 및 칸반 보드, 캘린더가 쾌적하게 보이도록 모바일 최적화 수정.

### B. 에러 핸들링 및 안정성 처리 (백엔드/프론트엔드)
1.  **전역 에러 바운더리(Error Boundary)**
    *   React 앱 전체가 충돌하는 것을 방지하기 위해 에러 바운더리 컴포넌트 추가.
2.  **API 에러 응답 통일성**
    *   백엔드 응답을 중앙집중식 에러 핸들러로 통과시켜 프론트엔드 단에서 일관된 에러 메시지(`react-hot-toast`) 표출.

### C. 배포 전략 수립 (Deployment)
1.  **프론트엔드 배포**: Vercel을 활용하여 `client` 폴더 배포 최적화.
2.  **백엔드 배포**: Railway 또는 Render를 통한 Express 서버 호스팅.
3.  **데이터베이스 연동**: 기존 개발용 Neon DB를 운영용으로 분리하거나 프로덕션 환경 변수(`.env`) 세팅 가이드라인 작성.

## 3. 사용자 피드백 반영 사항 (Confirmed)
*   **업무 드래그 앤 드롭**: 필수 기능으로 확정 적용. (리스트, 캘린더 등 모든 요소에 전면 적용)
*   **배포 환경**: Railway 서버리스 플랫폼 사용 확정.
    *   초기에는 Railway에서 제공하는 기본 무료 하위 도메인 사용 예정.
    *   새로운 프로젝트 생성 및 PostgreSQL(Neon 또는 Railway 내부 DB) 연결 필요.
