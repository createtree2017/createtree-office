# 플레이스 모니터링 확장 계획서

> **작성일**: 2026-03-11
> **분류**: Major Feature (≥ 200줄)
> **현 상태**: ✅ 네이버 플레이스 완료 / ❌ 구글·카카오 미지원
> **어제(3/10) 실패**: 카카오맵 비공식 API 3개 엔드포인트 전부 404/타임아웃 → 서버사이드 호출 차단 확인

---

## 1. 배경 및 문제 정의

### 현재 동작하는 것

- 네이버 플레이스 리뷰 수집 (GraphQL API) ✅
- 네이버 통합검색 모니터링 ✅
- 텔레그램 알림 시스템 ✅
- AI 감성 분석 + 보고서 생성 ✅

### 안 되는 것

- **카카오맵**: 비공식 API 차단 (2025.10~), axios 기반 수집 불가
- **구글 플레이스**: 미구현

### 사용자 요구사항

- 카카오맵이 **더 급함**
- 거래처에 비즈니스 프로필/카카오 비즈니스 등록 후 정보 전달을 요청할 수 있음
- Outscraper 유료 SaaS **사용 가능**

---

## 2. 우선순위 및 전략

```
1순위: 방안 A — Outscraper SaaS (구글 플레이스) → 가장 빠르고 확실
2순위: 방안 C — 카카오 비즈니스 파트너센터 자동화 (카카오맵) → 카카오가 급하지만 불확실성 있음
3순위: 방안 B — Google Business Profile API (구글 플레이스) → 무료 + 자동 답글 가능
```

> 카카오가 더 급하지만, 방안 A(Outscraper)는 2~3시간이면 끝나므로 먼저 빠르게 완료 후
> 방안 C(카카오)에 집중하는 것이 가장 효율적.

---

## 3. 방안별 상세 계획

### 방안 A: Outscraper SaaS (구글 플레이스)

**개발 범위**: 서버 수집기 1개 + monitoringService 분기 추가 + 프론트 플랫폼 선택지 추가

| 구분      | 내용                                               |
| --------- | -------------------------------------------------- |
| 소요 시간 | 2~3시간                                            |
| 비용      | Free tier: 월 100건 무료 / 이후 리뷰 1,000건당 ~$2 |
| 난이도    | ⭐ 낮음 (REST API 호출 + JSON 파싱)                |

**거래처에 받아야 할 정보:**

- ❌ 없음 — 구글 맵 URL만 있으면 됨 (예: `https://maps.google.com/maps/place/...`)
- 관리자 권한 불필요, 공개 리뷰 수집

**개발 항목:**

1. `server/src/services/monitoring/googlePlaceCollector.ts` [NEW]
   - Outscraper API 호출 (`GET https://api.app.outscraper.com/maps/reviews-v3`)
   - 응답 JSON → `PostData[]` 변환
2. `.env`에 `OUTSCRAPER_API_KEY` 추가
3. `monitoringService.ts`에 `platform === 'googleplace'` 분기 추가
4. `types.ts` platform/source 타입에 "googleplace" 추가
5. 프론트엔드 템플릿 생성 시 "구글 플레이스" 플랫폼 선택지 추가

**사용자 준비사항:**
| # | 할 일 | 방법 | 소요 |
|---|------|------|------|
| 1 | Outscraper 가입 | [outscraper.com](https://outscraper.com) 회원가입 | 3분 |
| 2 | API 키 발급 | Dashboard → API Keys → Create | 1분 |
| 3 | API 키 전달 | 발급받은 키를 저에게 전달 | - |

### 방안 B: Google Business Profile API (구글 플레이스 — 관리 매장)

**개발 범위**: OAuth 인증 + 리뷰 수집기 + DB 매핑

| 구분      | 내용                        |
| --------- | --------------------------- |
| 소요 시간 | 4~6시간                     |
| 비용      | 💰 무료                     |
| 난이도    | ⭐⭐ 중 (OAuth 인증)        |
| 제약      | 본인 관리 매장만            |
| 보너스    | ✨ 리뷰 자동 답글 기능 가능 |

**거래처에 받아야 할 정보:**

| #   | 필요 정보                            | 설명                                                                                   | 전달 방법          |
| --- | ------------------------------------ | -------------------------------------------------------------------------------------- | ------------------ |
| 1   | **구글 비즈니스 프로필 관리자 초대** | 거래처가 자기 매장의 Google Business Profile에서 대표님 계정(이메일)을 "관리자"로 추가 | 거래처가 직접 설정 |
| 2   | **매장 이름 + 주소**                 | 정확한 매장명과 주소 (API에서 매장 식별용)                                             | 텍스트 전달        |

> **거래처에 보낼 안내 문구 예시:**
>
> ```
> 안녕하세요, [거래처명]!
>
> 구글/카카오 리뷰 모니터링 서비스를 연동하려 합니다.
> 아래 절차를 진행해 주세요:
>
> 📌 구글 비즈니스 프로필:
> 1. Google 검색에서 "내 비즈니스" 검색 → 비즈니스 프로필 관리자 접속
> 2. 설정 → 사용자 → "9059056@gmail.com"을 관리자로 초대
> 3. 완료되면 알려주세요!
> ```

**개발 항목:**

1. `server/src/services/monitoring/googleBusinessCollector.ts` [NEW]
   - OAuth 2.0 인증 (기존 Google Drive 인증 모듈 재활용)
   - `accounts.locations.reviews.list` API 호출
2. DB: `clients` 테이블에 `google_account_id`, `google_location_id` 컬럼 추가
3. `monitoringService.ts` 분기 추가
4. 거래처 관리 UI에 구글 비즈니스 연동 상태 표시

**사용자 준비사항:**
| # | 할 일 | 방법 | 소요 |
|---|------|------|------|
| 1 | Google Cloud Console 프로젝트 확인 | [console.cloud.google.com](https://console.cloud.google.com) | 이미 있을 수 있음 |
| 2 | Business Profile API 활성화 | API 라이브러리 → "Business Profile API" 검색 → 활성화 | 2분 |
| 3 | OAuth 동의 화면 설정 | API → 사용자 인증 정보 → 내부 앱 | 10분 |
| 4 | 거래처에 관리자 초대 요청 | 위 안내 문구 전달 | 거래처마다 다름 |

### 방안 C: 카카오 비즈니스 파트너센터 자동화 (카카오맵)

**개발 범위**: Playwright 기반 로그인 + 리뷰 수집 자동화

| 구분      | 내용                                       |
| --------- | ------------------------------------------ |
| 소요 시간 | 8~12시간                                   |
| 비용      | 💰 무료 (서버 RAM 200~500MB 추가 필요)     |
| 난이도    | ⭐⭐⭐ 높음                                |
| 제약      | 본인 관리 매장만 / 카카오 계정 로그인 필요 |

**거래처에 받아야 할 정보:**

| #   | 필요 정보                                      | 설명                                                   | 전달 방법           |
| --- | ---------------------------------------------- | ------------------------------------------------------ | ------------------- |
| 1   | **카카오맵 매장 등록 확인**                    | 거래처 매장이 카카오맵에 사업자로 등록되어 있는지 확인 | 거래처에 확인 요청  |
| 2   | **카카오 비즈니스 파트너센터 관리자 추가**     | 파트너센터에서 대표님 카카오 계정을 관리자로 추가      | 거래처가 직접 설정  |
| 3   | **카카오 계정 로그인 정보** (또는 관리자 초대) | 자동화 로그인에 사용할 카카오 계정                     | 보안 정책 확인 필요 |

> **거래처에 보낼 안내 문구 예시:**
>
> ```
> 📌 카카오맵 리뷰 모니터링 연동:
> 1. 카카오맵에서 "장소 등록 요청" → 사업자인증으로 매장 등록 (이미 되어있으면 생략)
> 2. business.kakao.com 접속 → 매장 관리 → 관리자 초대
> 3. 2단계 인증 설정 여부 알려주세요
> ```

**⚠️ 진행 전 확인 필요사항 (대표님이 먼저 확인):**

1. [business.kakao.com](https://business.kakao.com) 또는 [partners.kakao.com](https://partners.kakao.com) 접속 → 매장 선택 → **리뷰 메뉴가 있는지** 스크린샷 공유
2. 2단계 인증 없이 로그인 가능한지 확인
3. 여러 거래처를 하나의 계정으로 관리 가능한지 확인

**개발 항목:**

1. `playwright` + `playwright-extra` + `puppeteer-extra-plugin-stealth` 패키지 설치
2. `server/src/services/monitoring/kakaoBusinessCrawler.ts` [NEW]
   - 카카오 로그인 자동화 (쿠키 저장/재사용으로 로그인 빈도 최소화)
   - 리뷰 목록 페이지 DOM 파싱 → `PostData[]` 변환
3. `monitoringService.ts` 분기 추가
4. `.env`에 `KAKAO_BIZ_EMAIL`, `KAKAO_BIZ_PASSWORD` (또는 토큰 기반)

## 4. 전체 개발 로드맵

```
Phase 1 (즉시 가능):
  ├─ 방안 A: Outscraper → API 키만 받으면 2~3시간 내 완료
  └─ 사용자: Outscraper 가입 + API 키 전달

Phase 2 (거래처 확인 후):
  ├─ 방안 C: 카카오 비즈니스 → 대시보드 리뷰 화면 확인 후 8~12시간
  └─ 사용자: 카카오 파트너센터 접속 → 리뷰 메뉴 스크린샷 공유

Phase 3 (선택):
  ├─ 방안 B: Google Business Profile API → 4~6시간
  └─ 사용자: 거래처에 관리자 초대 요청
```
