# Phase 2: 인플레이스 매뉴얼 에디터 계획서

## 1. 개요
*   **목적**: 사내 인수인계 매뉴얼을 별도의 관리자 페이지 없이 프론트엔드 사용자 화면에서 즉시 수정(In-place Editing)하고 자동 저장하는 기능을 구현함.
*   **핵심 가치**: 업무 흐름을 끊지 않는 빠른 문서화 및 최신 정보 유지.

## 2. 주요 기능 및 작업 내역
### 2.1. 데이터베이스 스키마 확장
*   `manuals` 테이블 생성: 제목, 내용(HTML/JSON), 작성자 ID, 카테고리, 버전 정보 포함.
*   `manual_versions` 테이블 (선택): 변경 이력 관리를 위한 스냅샷 저장.

### 2.2. 백엔드 API (Express)
*   매뉴얼 목록 조회 및 상세 조회 API.
*   매뉴얼 생성 및 수정을 위한 통합 저장 API (Upsert 로직).
*   이미지 업로드를 위한 엔드포인트 설계.

### 2.3. 프론트엔드 컴포넌트 (React + Tiptap)
*   `ManualList`: 카테고리별 매뉴얼 목록 네비게이션.
*   `EditableManual`: Tiptap 에디터를 활용한 본문 영역.
    *   읽기 모드: 텍스트 클릭 시 에디터 활성화.
    *   편집 모드: `onBlur` 시 자동 저장 통신 및 로딩 상태 표시.
*   TanStack Query를 이용한 서버 상태 관리 및 낙관적 업데이트(Optimistic Updates) 적용 고려.

## 3. 진행 일정
1.  DB 스키마 마이그레이션 (Drizzle): `manuals` 테이블에 편집 권한 등급 필드 추가.
2.  백엔드 CRUD API 구현 및 등급별 접근 제어(RBAC) 로직 적용.
3.  Tiptap 기반 상용 에디터 컴포넌트 구현 (`onBlur` 자동 저장 전용).
4.  관리자 전용 사용자 등급 관리 기능 구현.
5.  프론트-백엔드 연동 및 자동 저장 테스트.

## 4. 확정 사항 및 규칙
*   **저장 방식**: 별도의 저장 버튼 없이 `onBlur` 시 즉시 자동 저장 처리.
*   **권한 관리 (RBAC)**:
    *   **최고관리자 (ADMIN)**: 모든 설정 및 사용자 등급 변경 권한 보유.
    *   **관리자 (MANAGER)**: 허용된 범위 내 매뉴얼 수정 및 업무 관리.
    *   **직원 (USER)**: 기본적으로 읽기 권한, 특정 매뉴얼에 대해 편집 등급에 따라 수정 가능.
    *   매뉴얼별로 '수정 가능 최소 등급'을 지정하여 유연하게 운영.
