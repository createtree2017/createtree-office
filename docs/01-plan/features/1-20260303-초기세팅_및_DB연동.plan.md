# createTree Office 초기 세팅 및 DB 환경 구축 계획서

## 1. 개요
*   **목적**: 사내 전용 인수인계 및 매뉴얼 사이트 `createtree_office` 프로젝트의 로컬 개발 환경(프론트/백엔드)을 구축하고 전용 데이터베이스를 연결합니다.
*   **경로**: `C:\Users\TOP\Desktop\createAI\createtree_office`

## 2. 작업 내역 및 설계

### 2.1. 인프라 및 DB 세팅 (Neon)
기존 `createAI_v1` 데이터베이스와 완전히 분리된 데이터베이스가 필요합니다. 테스트와 실제 운영 데이터가 섞이지 않도록 새 프로젝트를 생성합니다.

1.  **Neon 콘솔 작업 (직접 진행 필요)**
    *   Neon 대시보드 로그인
    *   `New Project` (새 프로젝트 생성) 클릭
    *   프로젝트 이름: `createtree-office`
    *   Region: `Asia Pacific (Singapore)` 등 기존과 동일하게 설정
    *   생성된 `DATABASE_URL` 파악
2.  **환경 변수(.env) 설정**
    *   새로 발급받은 `DATABASE_URL`을 `.env`에 적용.
    *   사내 전용이므로 JWT Secret Key 등 백엔드 필수 환경변수 설정.

### 2.2. 프로젝트 스캐폴딩 (폴더/파일 초기화)
Full-stack Monorepo (client, server, shared 구조) 형태로 프로젝트를 구성합니다. 

1.  `npm init -y` 기반으로 메인 디렉토리 초기화
2.  **프론트엔드- [x] Project Scaffolding
    - [x] Initialize `package.json` (Root, Client, Server)
    - [x] Set up Backend (Express + Drizzle)
    - [x] Set up Frontend (Vite + React + Tailwind)
- [x] Scaffolding Complete
- [/] Handover
    - [ ] Inform user to switch to `createtree_office` workspace conversation
3.  **백엔드 (`server/`)**
    *   Express + TS 환경 세팅
    *   Drizzle ORM 세팅 및 기본 db 연결 테스트 
4.  **공통 모듈 (`shared/`)**
    *   API 스키마 및 DB 타입 공유용

### 2.3. 첫 번째 마이그레이션 (DB 스키마)
사내 직원 관리를 위한 최초의 테이블 모델링을 진행합니다.
*   `users` 테이블: 이름, 이메일, 비밀번호(해시), 승인 상태(`isApproved`), 역할(`role`: USER, ADMIN)

## 3. 진행 순서
1. Neon DB에서 새 프로젝트 생성 (USER)
2. 디렉토리 트리에 따른 npm 의존성 일괄 설치 및 스크립트 세팅 (AI)
3. `.env` 파일 구성 및 데이터베이스 접속 확인 (AI+USER)
4. React/Express "Hello World" 연동 테스트 (AI)
