# 모니터링 템플릿 분리(통합검색/플레이스) + 자동 스케줄링

## 배경
현재 모니터링 템플릿은 하나의 폼에서 통합검색(API)과 플레이스(크롤링)를 함께 처리.
플레이스 리뷰는 키워드가 불필요한데 키워드가 필수로 되어있어 UX 불편.
또한 자동화(매일 특정 시간 실행) 요구사항 추가.

## 변경 범위

### 1. DB Schema 수정 (`server/src/db/schema.ts`)
- `templateType` 컬럼 추가: `'integrated' | 'place'`
- `scheduleEnabled` 컬럼 추가: `boolean` (자동실행 활성화)
- `scheduleCron` 컬럼 추가: `text` (cron 표현식, 예: '0 9 * * *')
- `keywords` 컬럼을 nullable로 변경 (플레이스는 키워드 불필요)

### 2. 백엔드 서비스
#### `monitoringService.ts`
- `processMonitoring`: templateType에 따라 분기 처리
  - `integrated`: 기존 로직 (API + 크롤링)
  - `place`: URL 기반 리뷰 크롤링만 실행

#### `schedulerService.ts` [NEW]
- `node-cron` 설치 및 스케줄러 서비스 생성
- 서버 시작 시 활성화된 스케줄 등록
- 템플릿 CRUD 시 스케줄 동적 갱신

#### `monitoring.ts` (라우트)
- 템플릿 생성/수정 시 `templateType`, `scheduleEnabled`, `scheduleCron` 처리

### 3. 프론트엔드 UI (`MonitoringPage.tsx`)
- 템플릿 생성 시 **타입 선택 단계** 추가: [🔍 통합검색] / [🏥 플레이스]
- 통합검색 폼: 기존 폼 유지
- 플레이스 폼: 키워드 없이 URL + 수집 개수(10~100) + 거래처 선택
- 스케줄 설정 UI: 활성화 토글 + 실행 시간 선택
- 템플릿 목록에 스케줄/타입 정보 표시

## 검증 계획
- 로컬 빌드 테스트 (`npm run build`)
- 브라우저에서 통합검색/플레이스 템플릿 각각 생성 확인
- 스케줄 설정 후 콘솔 로그 확인
