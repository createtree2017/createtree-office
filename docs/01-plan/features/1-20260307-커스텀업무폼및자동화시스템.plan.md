# 창조트리 맞춤형 업무 폼 자동화 시스템 (Custom Task Form Automation) 기획서

## 1. 개요 및 목표
* **작성일**: 2026-03-07
* **목표**: 구글 폼의 형태적 한계와 API 연동의 복잡성을 배제하고, 창조트리 내부 시스템(DB)을 활용하여 **"업무 템플릿(질문지) 생성 -> 업무 배정 -> 작성 및 임시저장 -> 최종 완료 및 보고서 분석"**이 모두 사이트 내부에서 이루어지는 독자적인 B2B 워크플로우 시스템을 구축합니다.
* **핵심 이점**: 
  - 직원들은 개인 구글 계정 연동이나 구글 폼 화면 접속 없이 100% 사이트 내에서 매끄럽게 업무를 진행합니다.
  - 대용량 첨부 파일은 우리 서버를 거치지 않고(트래픽 최적화) 구글 드라이브 거래처 폴더로 다이렉트 업로드됩니다.
  - 향후 구글 캘린더 연동, 거래처 DB 연동 등 커스텀 확장성이 무한히 열립니다.

## 2. User Review Required (사용자 확인 필요 사항)
> [!IMPORTANT]
> **템플릿 빌더 구성 요소 한정**: 개발 효율성과 안정성을 위해, 관리자가 템플릿을 만들 때 사용할 수 있는 질문 유형은 다음 6가지 기능으로 한정하고 시작하는 것에 동의하십니까?
> 1. 단답형 (텍스트)
> 2. 장문형 (텍스트 에어리어)
> 3. 객관식 (라디오 버튼 - 단일 선택)
> 4. 다중 선택 (체크박스)
> 5. 드롭다운 선택
> 6. 파일 첨부 (Google Drive 다이렉트 업로드 전용)

## 3. 핵심 시스템 아키텍처 및 구현 모듈 (Proposed Changes)

### 3.1. DB 스키마 설계 (PostgreSQL)
구글 폼을 대체할 세 가지 메인 테이블이 추가됩니다.
* `task_templates` : 관리자가 만든 업무 템플릿 (양식 구조 JSON 저장)
* `task_instances` : 특정 거래처, 특정 월에 생성된 실제 업무 단위 (예: "3월 우성여성병원 서포터즈")
* `task_responses` : 직원이 입력 중인 임시 저장 데이터 및 최종 응답 데이터

### 3.2. 프론트엔드 (React) 모듈 개발
* **템플릿 빌더 UI (관리자용)**: 
  - 간단하게 문항(단답형, 객관식 등)을 추가하고 질문을 텍스트로 변경할 수 있는 UI.
* **커스텀 응답 UI (직원용 렌더러)**: 
  - `task_templates` 의 JSON 구조를 읽어와서 화면에 예쁜 폼 형태로 그려주는 공통 컴포넌트.
  - **자동/수동 임시 저장 기능** 포함.
* **Google Drive 다이렉트 업로더 (직원 첨부용)**: 
  - 구글 드라이브 `Resumable Upload` API를 활용하여 브라우저에서 구글 드라이브로 파일을 쏴버리는 최적화된 컴포넌트 (`<FileInput>`).
* **동적 보고서 대시보드 (통계)**: 
  - 완료된 `task_responses` 데이터를 긁어와서 표나 차트로 보여주는 리포트 페이지.

### 3.3. 백엔드 (Express) API 구조
* **템플릿 관리 API**: CRUD
* **업무 인스턴스 API**: 새 업무 생성 (구글 드라이브에 거래처/업무용 해당 하위 폴더 자동 생성 로직 포함).
* **업무 응답 API**: 임시 저장(Patch) 로직 및 최종 제출(Submit) 로직 처리.
* **업로드 티켓 API**: 프론트엔드가 파일을 직접 구글 드라이브로 보낼 수 있도록 `upload_id` (세션 URL)를 대신 발급받아 넘겨주는 API.

## 4. Verification Plan (검증 및 테스트 계획)

### 4.1. 수동 테스트 시나리오
1. **관리자 시점**: 사이트 내 `[템플릿 설정]`에서 "샘플 서포터즈" 양식을 정상적으로 6가지 유형으로 만들어 저장할 수 있는가?
2. **업무 배정 시점**: "우성여성병원"을 타겟으로 "3월 서포터즈" 업무를 생성했을 때, 연결된 구글 드라이브에 `우성여성병원/3월 서포터즈/` 폴더가 권한 설정 완료 상태로 즉시 생기는가?
3. **직원 진행 시점 (Draft)**: 직원이 1, 2번 문항만 작성하고 뒤로가기를 눌러도, 나중에 다시 들어왔을 때 내용이 그대로 살아있는가?
4. **파일 업로드 최적화 검증**: 50MB 이상의 영상을 첨부했을 때, 터미널(서버) 쪽에 부하(메모리 상승)나 로그 릴레이가 생기지 않고 브라우저단 네트워크로 바로 올라가는가? 그리고 그 파일이 2번에서 만든 폴더에 정확히 들어가는가?
5. **업무 완료 & 보고서**: 최종 제출 완료 시 데이터가 리포트 화면 포맷으로 이격 없이 노출되는가.
