# 계획서: 구글 드라이브 폴더 뷰어 연동 (B2B 파일 포털)

- **작성일**: 2026-03-06
- **상태**: 기획 (Plan)
- **목적**: 창조트리 사이트 내에서 지정된 구글 드라이브 폴더의 파일 목록을 조회하고 읽을 수 있는 통합 UI 구축

## 1. 업무 분담 (사용자 vs AI)

이 기능은 구글의 공식 API 서버를 사용해야 하므로, 철저한 권한 인증이 필요합니다. 따라서 개발 전 **관리자(사용자)님의 '구글 클라우드 보안 세팅'이 필수적**입니다.

### 👤 사용자(관리자님)가 해야 할 선행 작업
이 작업들은 제가 코드로 대신해 드릴 수 없으며, 관리자님이 직접 구글 사이트에 접속하셔서 버튼을 눌러 발급받아야 하는 "서버용 출입증(API 키)" 발급 과정입니다.

1. **Google Cloud Console 접속 및 프로젝트 생성**
   - 구글 클라우드 플랫폼(GCP)에 접속하여 새 프로젝트를 만듭니다.
2. **Google Drive API 활성화**
   - 해당 프로젝트에서 `Google Drive API`를 검색하여 '사용 설정'을 누릅니다.
3. **서비스 계정(Service Account) 생성**
   - 사용자 개인이 아닌, **우리 창조트리 서버가 사용할 가상의 구글 로봇 계정(이메일 형태)**을 만듭니다.
4. **보안 키(JSON) 다운로드**
   - 생성한 로봇 계정의 '키 탭'에서 새 키를 JSON 파일 형태로 다운로드합니다. 이 파일의 내용을 우리 프로젝트의 `.env` (환경변수)에 넣게 됩니다.
5. **열람할 폴더 공유 설정**
   - 구글 드라이브에서 사이트에 띄우고 싶은 폴더를 우클릭 -> 공유 -> **방금 만든 로봇 계정(서비스 계정) 이메일을 추가하고 '뷰어' 권한을 줍니다.**

### 🤖 AI(저)가 해야 할 개발 작업
관리자님이 시스템 출입증(JSON 키)을 발급해 오시면, 그 키를 들고 아래의 기능들을 개발합니다.

1. **백엔드 (API 연동)**
   - `googleapis` 패키지 설치
   - 서비스 계정 인증 로직 및 구글 드라이브 파일 목록 읽어오기(`GET /api/drive/folders/:id`) 라우터 개발
2. **프론트엔드 (UI 구축)**
   - 전달받은 파일 목록(이름, 썸네일, 링크, 수정일)을 창조트리 디자인(Tailwind 기반)에 맞게 표 또는 카드 UI로 제작
   - 엑셀, PDF, 이미지 등 클릭 시 구글의 공식 뷰어를 새 탭(또는 모달 미리보기)으로 띄워주는 로직 구현

## 2. 예상 구현 흐름도
1. `Client` -> `Server` : "A 회사의 보고서 폴더(ID: abc1234) 파일 목록 좀 줘"
2. `Server` -> `Google Drive API` : "나 창조트리 로봇인데(신분증 인증), abc1234 폴더 안에 있는 파일 리스트 내놔"
3. `Server` <- `Google Drive API` : "인증 확인됨. 여기 이름, 생성일, 뷰어 웹링크 JSON 데이터야"
4. `Client` <- `Server` : "받은 JSON 데이터" -> 예쁜 UI 렌더링 -> 클릭 시 뷰어 진입

### 2. 하위 폴더 통합 검색 기능 (추가 요청)
- **개요:** 현재 폴더 내에서만 필터링되던 로컬 검색 기능을 확장하여, 최상위 공유 폴더(`1SI_8POn6S3YqdEcrYIbFSzaU_r2fw5KI`) 하위의 모든 파일들을 재귀적으로(또는 드라이브 전체 범위에서 부모 폴더 제한 없이) 검색할 수 있도록 서버 사이드 API를 신설.
- **Backend (`server/src/routes/drive.ts`):** 
  - `GET /api/drive/search` 엔드포인트 추가
  - Google Drive API의 `q` 파라미터에 `name contains '검색어' and trashed = false` 조건을 부여하여 파일 검색 (특정 최상위 폴더 하위로 한정하는 로직 추가)
- **Frontend (`client/src/pages/DrivePage.tsx`):**
  - 검색어 입력 후 일정 시간(디바운스) 경과 시 또는 엔터 키 입력 시 `/api/drive/search` API 호출
  - 검색 중일 때는 폴더 탐색 구조를 무시하고 검색 결과 리스트만 화면에 렌더링하도록 뷰 전환 로직 적용
