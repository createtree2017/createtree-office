# O1-Plan: 거래처별 드라이브 폴더 매핑 및 권한 분리 시스템

## 작성일: 2026-03-06
## 목표: 
다중 테넌트(Multi-tenant) 구조를 차용하여 로그인된 사용자의 권한 및 거래처 소속에 따라 구글 드라이브 자료실의 접근 가능한 범위를 원천 분리한다.

## 상세 로직 설계
1. **역할(Role) 개편**: `ADMIN` (전체 제어), `MANAGER` (직원), `HOSPITAL_ADMIN` (거래처 원장), `USER` (가입증/권한대기자)
2. **거래처(Client) 테이블 및 매핑**:
   - `clients` 테이블 신설. 거래처명 및 해당 거래처의 전용 구글 루트 폴더 ID (`drive_folder_id`) 저장.
   - `users` 테이블에서 `client_id`를 외래키로 참조.
3. **가입 ও 승인 프로세스**:
   - 거래처 담당자는 최초 `USER` 로 가입하며 자료실 접근 불가.
   - 관리자가 거래처를 신설 시(구글 드라이브 폴더 API 동시생성) 해당 사용자에게 `HOSPITAL_ADMIN` 권한 부여 및 `client_id` 지정.
4. **동적 드라이브 라우팅**:
   - 자료실 접근 시, 로그인 정보를 바탕으로 루트 폴더(`currentFolder`)를 다르게 분기.
   - 백엔드에서도 사용자의 Role과 `clientId` 를 대조하여 요청된 폴더에 대한 권한 검증을 강제함.
