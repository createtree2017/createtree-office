# Phase 3: 업무 관리 (Todo & Calendar) 계획서

## 1. 개요
*   **목적**: 사내 직원의 업무 효율화를 위해 할 일(Todo) 목록과 일정(Calendar)을 통합 관리하는 시스템을 구축함.
*   **핵심 가치**: 업무 누락 방지, 일정 시각화, 담당자별 업무 최적화.

## 2. 주요 기능 및 작업 내역
### 2.1. 데이터베이스 스키마 확장
*   `tasks` 테이블 생성: 제목, 설명, 상태(대기/진행/보류/완료), 마감일, 담당자(assigneeId), 작성자(authorId) 포함.

### 2.2. 백엔드 API (Express)
*   도메인 분리 규칙에 따라 `server/src/routes/tasks.ts` 작성.
*   업무 생성 시 **담당자(Assignee) 지정 기능** 포함.
*   업무 상태 변경 및 필터링(내게 할당된 업무 등) API.

### 2.3. 프론트엔드 UI/UX (React)
*   **중앙집중식 모달**: `CreateTaskModal`을 통해 업무를 생성하고 다른 직원을 담당자로 지정.
*   **Todo 리스트**: '대기, 진행, 보류, 완료' 4단계 상태별 업무 관리.

## 3. 진행 일정
1.  DB 스키마 마이그레이션 (`tasks` 테이블 및 4단계 상태 Enum 추가).
2.  백엔드 Task API 구현 (업무 할당 로직 포함).
3.  직원 목록 조회를 통한 담당자 선택 UI 구현.
4.  Todo/Calendar 통합 뷰 개발.

## 4. 확정 사항
*   **업무 상태**: 대기(PENDING), 진행(IN_PROGRESS), 보류(ON_HOLD), 완료(COMPLETED) 4단계로 운영.
*   **핵심 기능**: 본인뿐만 아니라 **다른 직원에게 업무를 부여**하는 기능이 시스템의 핵심임.
