# Telegram 알림 시스템 계획서

> **작성일**: 2026-03-11
> **분류**: Major Feature (신규 기능)
> **PDCA 수준**: 필수

---

## 1. 목적

모니터링 완료, 업무 처리 완료 등 이벤트 발생 시 **거래처에게 Telegram 채팅 알림을 자동 발송**하는 시스템 구축.

### 왜 Telegram인가?

| 항목 | 설명                                       |
| ---- | ------------------------------------------ |
| 비용 | 완전 무료 (건당 과금 없음)                 |
| 구현 | API 코드 3줄 수준, 심사/승인 불필요        |
| 기능 | 마크다운, 버튼, 링크, 이미지 지원          |
| 조건 | 거래처가 Telegram 앱 설치 + 봇 "시작" 클릭 |

## 3. 구현 단계 (Phase별)

### Phase 1: 백엔드 핵심 — Telegram 연동 (우선)

1. Telegram Bot 생성 (BotFather) — 관리자 수동 작업
2. `telegramService.ts` 구현
   - 메시지 발송 (`sendMessage`)
   - Webhook 수신 (거래처 `/start` 감지 → chat_id 저장)
3. DB 스키마 변경: `clients` 테이블에 Telegram 필드 추가
4. `notification.ts` 라우트: Telegram webhook 수신 엔드포인트

### Phase 2: 모니터링 연동

1. `monitoringService.ts` 수정: 모니터링 완료 시 알림 발송
2. `monitoring_templates` 수정: `notify_enabled` 필드 추가
3. 모니터링 템플릿 UI에 알림 ON/OFF 토글 추가

### Phase 3: 거래처 관리 UI

1. 거래처 관리 화면에서 Telegram 연동 상태 표시
2. 초대 링크 생성 + 복사 기능
3. 알림 이력 조회

## 4. API 설계

### 4.1 Telegram Webhook 수신 (봇이 호출)

```
POST /api/notification/telegram/webhook
Body: Telegram Update 객체 (자동)
```

### 4.2 거래처 초대 링크 생성

```
GET /api/clients/:id/telegram-invite
Response: { inviteUrl: "https://t.me/createTreeBot?start=CODE123" }
```

### 4.3 알림 발송 테스트

```
POST /api/notification/test
Body: { clientId: 1, message: "테스트 메시지" }
```

### 4.4 알림 이력 조회

```
GET /api/notification/logs?clientId=1&page=1
```

## 5. 거래처 연동 플로우

```
[관리자] 거래처 관리에서 "초대 링크 생성" 클릭
    ↓
고유 코드 포함 링크 생성: https://t.me/createTreeBot?start=INV_거래처ID_랜덤코드
    ↓
[관리자] 거래처에게 링크 전달 (카톡, 문자, 이메일 등)
    ↓
[거래처] Telegram 앱에서 링크 클릭 → "시작" 버튼 누름
    ↓
[봇 Webhook] /start INV_거래처ID_랜덤코드 수신
    ↓
[서버] 코드로 거래처 식별 → chat_id 저장 → 환영 메시지 발송
    ↓
✅ 연동 완료 — 이후 자동 알림 발송 가능
```

## 7. 검증 계획

### 자동 테스트

- telegramService.ts 유닛 테스트 (메시지 포맷, API 호출 mock)

### 수동 테스트

1. BotFather로 테스트 봇 생성
2. 거래처 초대 링크 생성 → 본인이 직접 봇 시작 → chat_id 저장 확인
3. 모니터링 실행 → 완료 후 Telegram 메시지 수신 확인
4. 알림 이력 페이지에서 발송 기록 확인

### 확인 항목

- [ ] 거래처가 봇 시작 시 chat_id 자동 저장되는지
- [ ] 모니터링 완료 시 Telegram 알림이 발송되는지
- [ ] 알림 실패 시 에러가 기록되는지 (notification_logs)
- [ ] UI에서 연동 상태가 정확히 표시되는지
