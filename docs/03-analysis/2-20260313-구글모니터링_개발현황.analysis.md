## 3. 완료된 기능 목록

### ✅ 백엔드 — 완전히 작동하는 기능

1. **템플릿 CRUD** — 생성/조회/수정/삭제, 거래처별 분리
2. **통합검색 모니터링** — 네이버 API(블로그/카페/뉴스) + 콘텐츠 보강 크롤링
3. **네이버 플레이스 수집** — GraphQL API (pcmap-api.place.naver.com)
4. **카카오맵 리뷰 수집** — 비주얼 스크래핑 (Playwright + Gemini Vision)
5. **AI 감성 분석** — Gemini 2.5 Flash 일괄 분석 + 키워드 폴백
6. **보고서 생성** — HTML(구글 드라이브 자동 업로드) + 엑셀 다운로드
7. **자동 스케줄러** — setInterval 기반 cron 파싱, 30초 틱
8. **텔레그램 알림** — 모니터링 완료 시 알림 발송 (notifyEnabled 옵션)

### ✅ 프론트엔드 — 완전히 작동하는 기능

1. **대시보드 탭** — 총 템플릿, 분석 횟수, 수집 게시글, 감성 요약, 최근 활동
2. **템플릿 탭** — 목록 표시, 통합검색/플레이스 배지, 수동 실행, 자동 실행 토글
3. **결과 목록 탭** — 체크박스 다건 삭제, 상태 배지, 엑셀/보고서 버튼
4. **결과 상세 모달** — 감성 분포, AI 요약, 게시글 목록 (sentiment 뱃지)
5. **보고서 모달** — HTML 보고서 인앱 표시 + 새 탭 열기
6. **템플릿 생성/수정 폼** — 통합검색/플레이스 타입 분기, 키워드/플레이스 URL 입력, 스케줄 설정
7. **실행 중 자동 새로고침** — 5초 폴링, 결과 탭 30초 자동 새로고침

## 4. ❌ 미작동 — 구글 플레이스 리뷰 수집 상세 이슈

### 4.1 현재 구현 방식 (`visionCrawler.ts` → `captureGooglePlace`)

```
사용자 URL → 검색어(q=) 추출 → 구글 통합검색 접속 → "Google 리뷰" 링크 클릭
→ 리뷰 모달 오픈 → 스크롤 → 리뷰 영역 캡처 → Gemini Vision 분석
```

### 4.2 발생하는 문제점

| # | 이슈 | 상세 |
|---|------|------|
| 1 | **Headless 감지/차단** | 구글은 Headless Chromium 접속을 감지하여 CAPTCHA나 제한된 뷰를 표시. Stealth 스크립트(webdriver 우회, Chrome 런타임 시뮬레이션 등)를 적용했지만 불충분 |
| 2 | **리뷰 모달 미오픈** | `a:has-text("Google 리뷰")` 링크가 통합검색 결과에서 항상 나오지 않거나, 클릭해도 모달이 안 열림 |
| 3 | **CSS 셀렉터 불일치** | 리뷰 컨테이너 셀렉터(`div.review-dialog-list`, `div.gws-localreviews__general-reviews-block` 등)가 구글의 동적 DOM 구조와 맞지 않아 캡처 실패 |
| 4 | **폴백 캡처 품질 저하** | 리뷰 컨테이너를 못 찾으면 전체 페이지를 캡처하지만, 검색 결과가 표시되어 리뷰 추출이 안 됨 |
| 5 | **쿠키 동의 팝업** | 구글 쿠키 동의 팝업이 표시되어 리뷰 접근이 차단될 수 있음 (팝업 닫기 로직 존재하지만 동작 불안정) |

### 4.3 이전에 시도한 방법들 (시간 순)

1. **구글 맵 직접 접속 (처음)** → Headless 감지로 "제한된 뷰" 표시 → 실패
2. **구글 통합검색 방식 전환 (2026-03-12)** → URL에서 `q=` 파라미터 추출 → `google.com/search?q={검색어}` 접속 → 리뷰 링크 클릭 시도 → 모달/셀렉터 문제로 실패
3. **세션 토큰 제거 (클린 URL 생성)** → `sxsrf`, `si`, `ved` 등 세션 파라미터 제거 → 일부 개선되었으나 근본적 미해결
4. **Stealth 스크립트 강화** → `navigator.webdriver` 제거, Chrome 런타임 시뮬레이션, WebGL 위장 → 여전히 감지됨

### 4.4 `googlePlaceCollector.ts` (Outscraper API) — 폴백 후보

- Outscraper SaaS API (`api.app.outscraper.com`)를 통해 구글 맵 리뷰를 수집하는 코드가 이미 `googlePlaceCollector.ts`에 구현되어 있음
- **그러나 현재 `monitoringService.ts`에서 호출되지 않고 있음**
- 구글 플레이스를 `visionCrawler`만 사용하도록 라우팅되어 있음 (L142: `place.platform === 'googleplace'` → `visionCrawler.crawlByVision()`)
- `.env`에 `OUTSCRAPER_API_KEY`가 설정되어 있으나 사용되지 않는 상태

## 6. 향후 수정 필요 사항 — 우선순위순

### 🔴 P0: 구글 플레이스 리뷰 수집 해결 (핵심 미작동 이슈)

**선택지 A: Outscraper API 폴백 활성화 (빠른 해결)**

```
변경 위치: monitoringService.ts (L142 부근)
변경 내용:
  1. googleplace일 때 먼저 visionCrawler로 시도
  2. 실패 시(빈 배열) → googlePlaceCollector.crawlGooglePlace()로 폴백
  3. 또는: googleplace만 아예 Outscraper API로 고정
```

- 장점: 안정적, 봇 감지 없음, API 기반
- 단점: 유료 API (Outscraper 크레딧 소모), 매월 비용 발생 가능
- 주의: `OUTSCRAPER_API_KEY`는 이미 `.env`에 설정되어 있으므로 코드 변경만 필요

**선택지 B: Playwright Stealth 강화 + 캡처 전략 재설계 (시간 소요)**

```
검토할 부분:
  1. playwright-extra + puppeteer-extra-plugin-stealth 포트 사용
  2. 구글 맵 접속 대신 Google Maps Reviews 전용 URL 패턴 탐색
  3. 비 headless 모드 테스트 (headless: false)로 실제 동작 확인
  4. 실시간 DOM 구조 분석 → CSS 셀렉터 업데이트
  5. IP 우회 (프록시) 검토
```

- 장점: 무료, API 의존 없음
- 단점: 구글의 봇 감지가 매우 강력하여 지속적 유지보수 필요, 불안정

**선택지 C: Google Places API (정식 API)**

```
Google Cloud Console에서 Places API 활성화
→ Place Details API로 리뷰 수집
→ 제한: 최대 5개 리뷰만 반환 (최신 순)
```

- 장점: 공식 API, 안정적
- 단점: 리뷰 최대 5개만 반환, 유료 (Monthly $200 크레딧 있음)

## 7. 실행 흐름 요약 (코드 레벨)

### 모니터링 실행 시 (`POST /api/monitoring/templates/:id/execute`)

```
1. monitoring.ts 라우트 → monitoringService.executeMonitoring(templateId, userId)
2. DB에 결과 레코드 생성 (status: "RUNNING")
3. 비동기로 processMonitoring() 실행
4. processMonitoring() 내부:
   a. templateType === 'place' 분기:
      - platform === 'kakaomap' || 'googleplace' → visionCrawler.crawlByVision()
      - platform === 'naverplace' → placeCrawler.crawlNaverPlace()
   b. templateType === 'integrated' 분기:
      - naverCollector.crawlSearch() → contentCrawler.enrichPosts()
5. AI 감성 분석: sentimentAnalyzer.analyze(posts)
6. 보고서 생성 + 드라이브 업로드: reportGenerator.generateAndUpload()
7. DB 결과 업데이트 (status: "COMPLETED")
8. 텔레그램 알림: telegramService.sendMonitoringAlert()
```

### VisionCrawler 구글 캡처 흐름 (`captureGooglePlace`)

```
1. URL에서 q= 파라미터 추출 또는 /maps/place/{장소명} 추출
2. 구글 통합검색 URL 생성: google.com/search?q={검색어}&hl=ko
3. Playwright headless Chromium 실행 (stealth 스크립트 주입)
4. 구글 검색 페이지 접속
5. 쿠키 동의 팝업 닫기 시도
6. "Google 리뷰" 링크 클릭 → 리뷰 모달 오픈 시도 ← ⚠️ 여기서 실패
7. 정렬 변경 (최신순) 시도
8. 리뷰 모달 내부 스크롤 시도
9. 리뷰 컨테이너 셀렉터로 영역 캡처 시도 ← ⚠️ 셀렉터 불일치
10. 폴백: 전체 페이지 캡처 → Gemini Vision에 전달
11. Gemini Vision이 리뷰 데이터 없이 빈 배열 반환
```
