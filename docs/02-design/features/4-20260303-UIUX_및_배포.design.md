# 설계 문서: UI/UX 고도화 및 배포 준비 (Phase 4)

## 1. 개요
*   **작성일**: 2026-03-03
*   **기능명**: UI/UX Drag & Drop 고도화 및 Railway 배포
*   **목적**: 업무 관리 화면(보드, 캘린더)에 Drag & Drop 인터랙션을 전면 적용하여 사용성을 극대화하고, 모바일 반응형 최적화 후 Railway에 시스템을 호스팅합니다.

## 2. 프론트엔드 설계 (UI/UX)

### 2.1 Drag and Drop (DnD) 시스템 적용
*   **라이브러리**: `@hello-pangea/dnd` (React 18 호환성이 보장되는 react-beautiful-dnd의 포크 버전)
*   **적용 대상**:
    1.  **Kanban Board (`TasksPage.tsx`)**:
        *   `DragDropContext`: 전체 보드 컨테이너
        *   `Droppable`: 4개의 각 상태별 칼럼(대기, 진행, 보류, 완료)
        *   `Draggable`: 개별 업무(Task) 카드 컴포넌트
        *   *동작 로직*: 카드를 다른 칼럼으로 드롭 시 즉각 프론트엔드 상태를 업데이트(`onDragEnd` 핸들러)하고, 백엔드 `PATCH /api/tasks/:id` API를 호출하여 상태를 영구 저장.
    2.  **Calendar View (`TasksPage.tsx`)**:
        *   일정 리스트를 드래그하여 날짜를 옮기거나 순서를 변경하는 로직 추가. (구현 복잡도에 따라 달력 인터페이스 호환성 별도 검토)

### 2.2 반응형(Responsive) 및 로딩(Loading) UI
*   **Kanban Board**: 모바일 화면(width < 768px)에서는 각 상태 칼럼이 가로로 스크롤(Snap Scroll)되거나 위아래로 나열되도록 Tailwind CSS 분기 처리.
*   **Global Loading**: 데이터 패치(TanStack Query) 중일 때 빈 화면이 아닌 모듈식 스피너 또는 골격(Skeleton) 노출.

## 3. 백엔드 설계 및 배포(Deployment) 전략

### 3.1 Railway 호스팅 스키마
1.  **DB 구성**: 개발 환경의 Neon DB 환경변수(`DATABASE_URL`)를 Railway 프로젝트에 등록하여 그대로 연동.
2.  **프로젝트 구조 (Monorepo 지원)**:
    *   Railway에서 `npm run build` 스크립트를 통해 프론트엔드를 빌드.
    *   Express 서버에서 프론트엔드의 `dist/` 폴더를 정적으로 서빙(Static Serve)하는 단일 서버 아키텍처로 구성.
3.  **환경변수 (Environment Variables)**:
    *   `PORT`: Railway에서 할당하는 자동 포트 사용 (`process.env.PORT`)
    *   `JWT_SECRET`: 프로덕션 레벨 시크릿 키로 재설정.
    *   `DATABASE_URL`: 기존 Neon DB 연결 문자열 복사.
