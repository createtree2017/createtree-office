# Phase 2: 인플레이스 매뉴얼 에디터 설계서

## 1. 데이터 모델 설계 (Drizzle ORM)

### `manuals` 테이블
```typescript
export const manuals = pgTable("manuals", {
  id: serial("id").primaryKey(),
  title: text("title").notNull(),
  content: text("content").notNull(), 
  categoryId: integer("category_id"),
  authorId: integer("author_id").references(() => users.id),
  minRoleToEdit: userRoleEnum("min_role_to_edit").default("MANAGER").notNull(), // 편집 가능 최소 등급
  version: integer("version").default(1).notNull(),
  createdAt: timestamp("created_at").defaultNow().notNull(),
  updatedAt: timestamp("updated_at").defaultNow().notNull(),
});
```

## 2. API 엔드포인트 설계

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| GET | `/api/manuals` | 매뉴얼 목록 조회 | Approved User |
| GET | `/api/manuals/:id` | 특정 매뉴얼 상세 조회 | Approved User |
| POST | `/api/manuals` | 새 매뉴얼 생성 | Manager/Admin |
| PATCH | `/api/manuals/:id` | 매뉴얼 내용 수정 | minRoleToEdit 확인 로직 포함 |
| PATCH | `/api/admin/users/:id/role` | 사용자 등급 변경 | Admin Only |

## 3. 프론트엔드 아키텍처

### 인터랙션 흐름
1.  사용자가 매뉴얼 본문 영역 클릭.
2.  `isEditing` 상태 true 전환 -> Tiptap 에디터 인스턴스 활성화.
3.  내용 수정 후 에디터 외부 클릭 (`onBlur`).
4.  `useMutation` (TanStack Query) 트리거 -> 백엔드 PATCH 요청.
5.  저장 완료 시 토스트 알림 또는 우측 상단 '저장됨' 표시.

### 컴포넌트 구조
- `ManualPage`: 전체 페이지 레이아웃 및 데이터 패칭.
- `ManualSidebar`: 카테고리 트리 UI.
- `ManualContent`: 제목 및 에디터 영역.
  - `TitleSection`: 인라인 제목 수정.
  - `EditorSection`: Tiptap 커스텀 확장 기능(Bold, Italic, List, Image) 포함.

## 4. 예외 처리
- **동시 수정 충돌**: 작업 도중 다른 사람이 수정했을 경우 경고 메시지 표시 및 최신 버전 반영 제안.
- **네트워크 오류**: 자동 저장 실패 시 로컬 스토리지에 임시 저장 후 재연결 시 시도.
