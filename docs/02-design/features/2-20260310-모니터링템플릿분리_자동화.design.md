# 모니터링 템플릿 분리 및 자동화 설계

> 작성일: 2026-03-10 | 상태: 구현 완료

## 1. 개요

모니터링 시스템의 템플릿을 **통합검색**과 **플레이스** 두 가지 타입으로 분리하고,
`node-cron` 기반 자동 스케줄링 기능을 추가합니다.

## 2. 변경 사항

### 2.1 DB 스키마 (`server/src/db/schema.ts`)
- `templateType` 컬럼 추가 (`integrated` | `place`, 기본값: `integrated`)
- `keywords` nullable로 변경 (플레이스는 키워드 불필요)
- `scheduleEnabled`, `scheduleCron`, `scheduleLastRunAt` 컬럼 추가

### 2.2 타입 (`server/src/services/monitoring/types.ts`)
- `MonitoringTemplate` 인터페이스에 위 필드 추가

### 2.3 스케줄러 (`server/src/services/monitoring/schedulerService.ts`) [신규]
- `node-cron` 기반 `SchedulerService` 클래스
- 서버 시작 시 DB에서 활성 스케줄 로드 (`initializeSchedules`)
- 템플릿 생성/수정/삭제 시 스케줄 등록/해제 (`registerSchedule`, `removeSchedule`)

### 2.4 라우트 (`server/src/routes/monitoring.ts`)
- POST/PUT `/templates`: `templateType`, `scheduleEnabled`, `scheduleCron` 처리
- 키워드 유효성: 통합검색만 필수
- 스케줄 서비스 연동

### 2.5 서버 진입점 (`server/src/index.ts`)
- 서버 시작 시 `SchedulerService.initializeSchedules()` 호출

### 2.6 프론트엔드 (`client/src/pages/MonitoringPage.tsx`)
- **Template 인터페이스**: `templateType`, `scheduleEnabled`, `scheduleCron` 등 추가
- **템플릿 목록**: 타입 배지(🔍/🏥), 스케줄 배지(🕐) 표시
- **TemplateFormModal**: 2단계 폼 (타입선택 → 세부설정)
  - 통합검색: 기존 키워드/범위/수집방식 폼
  - 플레이스: URL 입력 (네이버/카카오/구글) + 수집개수
  - 공통: 수집개수 슬라이더, 자동실행 토글+시간 선택

### 2.7 인프라
- `package.json`: `node-cron` 의존성, `engines.node >= 20`
- `nixpacks.toml`: Chromium 설치 설정

## 3. 플레이스 모니터링 흐름

```
[사용자] → [템플릿 생성 (place 타입)]
  ↓ URL 입력 (네이버/카카오/구글 플레이스)
  ↓ 수집개수 설정 (최대 100개)
  ↓ 자동실행 설정 (선택)
[실행] → Puppeteer로 해당 URL 크롤링
  ↓ 최신순 리뷰 수집 (지정 개수)
  ↓ AI 감성 분석
[결과] → 보고서 생성
```

## 4. 자동 스케줄링 흐름

```
[서버 시작] → SchedulerService.initializeSchedules()
  ↓ DB에서 scheduleEnabled=true 템플릿 조회
  ↓ 각 템플릿의 scheduleCron에 따라 cron job 등록
[매일 HH:00] → 자동으로 모니터링 실행
  ↓ scheduleLastRunAt 업데이트
```
