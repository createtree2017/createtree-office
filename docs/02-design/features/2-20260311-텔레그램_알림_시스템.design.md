## 1. DB 스키마 설계

### 1.1 clients 테이블 변경 (ALTER)

```sql
ALTER TABLE clients ADD COLUMN telegram_chat_id TEXT;
ALTER TABLE clients ADD COLUMN telegram_invite_code TEXT;
ALTER TABLE clients ADD COLUMN telegram_connected_at TIMESTAMP;
```

| 필드                    | 타입      | 설명                                      |
| ----------------------- | --------- | ----------------------------------------- |
| `telegram_chat_id`      | TEXT      | Telegram 대화방 ID (봇 시작 시 자동 저장) |
| `telegram_invite_code`  | TEXT      | 고유 초대 코드 (거래처 식별용)            |
| `telegram_connected_at` | TIMESTAMP | 연동 완료 시각                            |

### 1.2 monitoring_templates 변경 (ALTER)

```sql
ALTER TABLE monitoring_templates ADD COLUMN notify_enabled BOOLEAN DEFAULT false;
ALTER TABLE monitoring_templates ADD COLUMN notify_channels JSONB DEFAULT '["telegram"]';
```

### 1.3 notification_logs 테이블 (CREATE)

```sql
CREATE TABLE notification_logs (
  id SERIAL PRIMARY KEY,
  client_id INTEGER REFERENCES clients(id),
  channel TEXT NOT NULL DEFAULT 'telegram',          -- telegram | email | ...
  message_type TEXT NOT NULL DEFAULT 'monitoring',   -- monitoring | task | custom
  content TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'sent',               -- sent | failed
  error_message TEXT,
  template_id INTEGER REFERENCES monitoring_templates(id),
  result_id INTEGER REFERENCES monitoring_results(id),
  created_at TIMESTAMP DEFAULT NOW() NOT NULL
);
```

## 2. 서비스 설계

### 2.1 TelegramService (server/src/services/notification/telegramService.ts)

```typescript
class TelegramService {
  private botToken: string;

  // 메시지 발송
  async sendMessage(
    chatId: string,
    text: string,
    parseMode?: string,
  ): Promise<boolean>;

  // 모니터링 완료 알림 발송 (포맷팅 포함)
  async sendMonitoringAlert(
    client: Client,
    result: MonitoringResult,
  ): Promise<boolean>;

  // Webhook 처리: /start 명령 → chat_id 저장
  async handleWebhook(update: TelegramUpdate): Promise<void>;

  // 초대 코드 생성
  generateInviteCode(clientId: number): string;

  // 초대 URL 생성
  getInviteUrl(inviteCode: string): string;
}
```

### 2.2 Webhook 처리 플로우

```
Telegram → POST /api/notification/telegram/webhook
  ↓
  message.text === "/start INV_1_ABC123"
  ↓
  DB에서 invite_code "INV_1_ABC123" 검색 → 거래처 찾기
  ↓
  clients.telegram_chat_id = message.chat.id 업데이트
  ↓
  환영 메시지 발송: "✅ 그레이스병원 알림 연동 완료!"
```

## 3. API 엔드포인트 설계

### 3.1 notification.ts 라우트

| Method | Path                                 | 설명                         |
| ------ | ------------------------------------ | ---------------------------- |
| POST   | `/api/notification/telegram/webhook` | Telegram Webhook (봇 → 서버) |
| POST   | `/api/notification/test`             | 테스트 알림 발송             |
| GET    | `/api/notification/logs`             | 알림 이력 조회               |

| Method | Path                               | 설명                |
| ------ | ---------------------------------- | ------------------- |

| DELETE | `/api/clients/:id/telegram`        | Telegram 연동 해제  |

## 4. 프론트엔드 UI 설계

### 4.1 모니터링 템플릿 모달 — 알림 섹션 추가

기존 `MonitoringPage.tsx`의 TemplateFormModal에 아래 섹션 추가:

```
┌─────────────────────────────────────┐
│ 🔔 완료 알림                         │
│                                     │
│ Telegram 알림 발송   [ON/OFF 토글]    │
│                                     │
│ (거래처 연동 상태 표시)               │
│ ✅ 그레이스병원 — Telegram 연동됨     │
│ 또는                                │
│ ⚠️ 그레이스병원 — Telegram 미연동     │
│ [초대 링크 보내기]                    │
└─────────────────────────────────────┘
```

### 4.2 거래처 관리 — Telegram 컬럼

기존 거래처 목록에 Telegram 연동 상태 표시:

- ✅ 연동됨 (connected_at 표시)
- ❌ 미연동 (초대 링크 생성 버튼)
- 🔄 대기중 (초대 코드 생성됨, 아직 시작 안 함)
