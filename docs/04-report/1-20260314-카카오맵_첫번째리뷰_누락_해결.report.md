# 카카오맵 첫 번째 리뷰 누락 문제 해결 리포트

> **작성일**: 2026-03-14  
> **수정 파일**: `server/src/services/monitoring/visionCrawler.ts`  
> **상태**: ✅ 해결 완료

---

## 1. 문제 현상

카카오맵 플레이스 모니터링 실행 시, **첫 번째 리뷰가 항상 누락**되고 두 번째 리뷰부터 결과에 포함됨.

- 정렬 순서(최신순/유용한순)와 관계없이 발생
- 첫 번째 리뷰 작성자를 변경해도 동일하게 **항상 첫 번째만 누락**
- 10건 수집 시 실제로는 2번째~11번째 리뷰가 수집됨

---

## 2. 시도한 해결 방법과 결과

### ❌ 시도 1: Gemini Vision 프롬프트 강화
- **가설**: AI가 첫 번째 리뷰를 UI 헤더로 오인하여 건너뛴다
- **조치**: 프롬프트에 "첫 번째 리뷰를 절대 건너뛰지 마라", `position` 필드 추가
- **결과**: 실패 — 프롬프트 문제가 아니라 **이미지 자체에 첫 번째 리뷰가 없었음**

### ❌ 시도 2: viewport를 500px 모바일로 변경 + iPhone UA 적용
- **가설**: 카카오맵이 모바일 최적화 사이트라 데스크톱 1920px에서 레이아웃이 깨진다
- **조치**: `viewport: 500px`, `userAgent: iPhone Safari`로 변경
- **결과**: 실패 — **0건 결과**. 모바일 viewport에서 DOM 구조가 달라져 셀렉터 매칭 실패. 카카오맵 place 페이지는 PC/모바일 동일 레이아웃이므로 viewport 변경 불필요

### ❌ 시도 3: 정렬 드롭다운 오버랩 방지
- **가설**: "최신 순/유용한 순" 드롭다운이 열릴 때 첫 번째 리뷰를 물리적으로 가린다
- **조치**: 
  - 이미 "최신 순"이면 드롭다운 열지 않음
  - 드롭다운 실패 시 `body.click()` → `Escape` 키로 닫기
- **결과**: 부분적 발견 — 드롭다운이 열리면 실제로 첫 번째 리뷰를 가리는 것은 맞지만, **정렬 로직을 완전 비활성화해도 문제 동일**. 드롭다운은 원인이 아니었음

> ⚠️ `body.click({x:10, y:10})` 사용 시, 카카오맵 상단의 햄버거 메뉴(≡)를 클릭하여 **페이지 이탈 → 0건 결과** 발생 주의

### ❌ 시도 4: 고정 헤더 CSS display:none 숨기기
- **가설**: 고정 탭 메뉴(홈/사진/후기 등)가 element screenshot에서 첫 번째 리뷰를 가린다
- **조치**: `page.evaluate()`로 `.view_atab, .head_place` 등 CSS `display: none` 처리
- **결과**: 실패 — CSS 셀렉터가 실제 카카오맵 DOM과 매칭되지 않아 헤더가 숨겨지지 않음

### ✅ 시도 5: fullPage screenshot으로 전환 (최종 해결)
- **가설**: element screenshot(`.list_review`) 자체가 고정 헤더에 의해 첫 번째 리뷰 영역이 가려진 상태로 캡처된다
- **조치**: `.list_review` element screenshot → `page.screenshot({ fullPage: true })` 전환
- **결과**: **성공!** 첫 번째 리뷰(뿌루조아)가 정상적으로 #1에 포함됨

## 3. 근본 원인 분석

```
카카오맵 리뷰 페이지 구조:

┌──────────────────────────┐
│ kakaomap 헤더             │  ← position: fixed/sticky
├──────────────────────────┤
│ 탭 메뉴 (홈/사진/후기...) │  ← position: sticky (스크롤 시 상단 고정)
├──────────────────────────┤
│ 정렬 버튼 (최신순/유용한순) │
├──────────────────────────┤
│ ██ 첫 번째 리뷰 (가려짐) ██ │  ← 고정 헤더 뒤에 숨겨짐!
│ ── 두 번째 리뷰 ──────── │
│ ── 세 번째 리뷰 ──────── │
│ ...                      │
└──────────────────────────┘
```

1. URL에 `#comment` 해시로 접속 → 브라우저가 리뷰 섹션으로 **자동 스크롤**
2. 상단 탭 메뉴가 `position: sticky`로 **고정**되어 스크롤해도 사라지지 않음
3. element screenshot(`.list_review`)은 해당 요소의 **렌더링된 위치** 기준으로 캡처
4. 고정 헤더가 첫 번째 리뷰의 **작성자명/날짜/별점 영역을 덮은 상태**로 캡처됨
5. Gemini AI가 작성자 정보 없는 이미지 조각(사진만)을 **리뷰로 인식하지 못함**
6. 결과적으로 **두 번째 리뷰부터** 추출됨

## 4. 최종 해결 코드

```typescript
// visionCrawler.ts - captureKakaoMap 함수 내

// ★ 카카오맵: fullPage screenshot 사용
// element screenshot은 고정 탭 헤더가 첫 번째 리뷰를 가리므로

const screenshot = await page.screenshot({
    type: "png",
    fullPage: true,
});
return Buffer.from(screenshot);
```

## 5. 핵심 교훈

| # | 교훈 |
|---|------|
| 1 | **element screenshot은 sticky/fixed 헤더에 취약** — 요소 자체는 정상이어도, 렌더링 시 고정 요소가 위를 덮으면 캡처에 포함됨 |
| 2 | **카카오맵 place 페이지는 PC/모바일 동일 레이아웃** — viewport/UA 변경 불필요, 오히려 DOM 구조가 달라져 셀렉터 실패 유발 |
| 3 | **`body.click()` 좌표(10,10)은 위험** — 상단에 내비게이션 링크가 있으면 페이지 이탈 |
| 4 | **CSS display:none은 정확한 셀렉터 필요** — 브라우저 DevTools에서 사전 확인 필수 |
| 5 | **디버그 스크린샷은 fullPage로** — element screenshot과 다른 결과를 보여줘 혼동 유발. 실제 Gemini에 전달되는 이미지와 동일한 방식으로 저장해야 함 |

## 6. 향후 유사 문제 발생 시 체크리스트

1. **디버그 스크린샷 확인**: `/tmp/debug_kakaomap_*.png` 파일에서 첫 번째 리뷰가 온전히 포함되어 있는지 확인
2. **fullPage vs element**: 고정 헤더가 있는 사이트는 fullPage screenshot 우선 사용
3. **정렬 드롭다운**: 이미 원하는 정렬이면 드롭다운 열지 않기 (드롭다운도 리뷰를 가릴 수 있음)
4. **이미지 크기**: fullPage screenshot은 파일이 클 수 있으므로 `maxReviews`에 맞는 viewport 높이 설정 중요
