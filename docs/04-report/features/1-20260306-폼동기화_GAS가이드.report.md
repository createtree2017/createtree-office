## 1. 구축 완료된 사항

- **Neon DB 구조 확장**: 구글 폼 응답 데이터(질문 항목들과 그에 대한 응답) 전체를 안전하고 유연하게 저장하기 위한 `form_submissions` 테이블과 `jsonb` 데이터 타입 정의를 Drizzle 스키마에 추가하고 마이그레이션을 완료했습니다.
- **Express 백엔드 웹훅**: 누군가 폼을 제출하는 즉시 구글이 창조트리에 쏠 신호를 받을 수 있도록 `POST /api/webhook/google-sync` 라우터를 개발하고 메인 서버 통신망(`index.ts`)에 연결했습니다. 보안을 위한 시크릿 키 검증 로직도 포함되었습니다.

## 2. 관리자님, 이 설정을 진행해 주세요! (구글 시트 연동)

완전한 연동을 위해 데이터의 시작점인 **'해당 폼과 연결된 구글 시트'**에서 데이터를 쏴주는 스크립트를 설정해야 합니다.

### [1단계] 스크립트 붙여넣기
1. 알려주신 폼과 연결된 구글 스프레드시트 창을 엽니다.
2. 상단 메뉴에서 **[확장 프로그램] -> [Apps Script]**를 클릭합니다.
3. 열린 창의 기존 내용(`function myFunction() {...}`)을 전부 지우고, 아래의 전체 코드를 복사해서 붙여넣습니다.

```javascript
/* 창조트리 웹훅 전송 스크립트 */
var WEBHOOK_URL = "여기에_배포된_서버주소/api/webhook/google-sync"; 
// ※ 주의: 로컬 테스트(npm run dev) 중일 때는 ngrok 등을 사용한 외부 주소가 필요합니다.
var SECRET_KEY = "createtree-sync-key"; // 서버와 약속한 보안 키
var FORM_ID = "1RuM8pkO0yZQc9jii_InCe0-ovyBXW5VtvPXl7NEJBoc"; // 전달받은 폼 ID

function onFormSubmitTrigger(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var range = e.range;
  var rowIndex = range.getRow();
  
  var formData = {};
  
  // 구글 폼에서 넘어온 질문 텍스트(Header)와 사용자가 선택한 답변 매핑
  if (e.namedValues) {
    for (var key in e.namedValues) {
      if (e.namedValues[key] && e.namedValues[key].length > 0) {
        formData[key] = e.namedValues[key][0];
      }
    }
  }

  // 창조트리 서버로 쏠 데이터 패키징
  var payload = {
    "secret": SECRET_KEY,
    "formId": FORM_ID,
    "rowIndex": rowIndex,
    "timestamp": new Date().toISOString(),
    "data": formData
  };

  var options = {
    "method": "post",
    "contentType": "application/json",
    "payload": JSON.stringify(payload)
  };

  try {
    // 백엔드로 발사!
    UrlFetchApp.fetch(WEBHOOK_URL, options);
  } catch (error) {
    Logger.log("Webhook 발송 실패: " + error.toString());
  }
}
```

### [2단계] 자동 실행 (트리거) 달아주기
코드가 붙여넣어 졌다면, 누군가 폼을 '제출(Submit)'할 때마다 위 스크립트가 실행되게 해야 합니다.

1. 스크립트 화면 왼쪽 도구 모음에서 **[시계 모양 아이콘(트리거)]** 선택
2. 우측 하단의 파란색 **[+ 트리거 추가]** 버튼 클릭
3. 아래와 같이 설정하고 **[저장]** 버튼을 누릅니다.
   - 실행할 함수: `onFormSubmitTrigger`
   - 이벤트 소스: `스프레드시트에서`
   - 이벤트 유형: `양식 제출 시`
4. *(중요)* 저장 시 구글 계정 로그인 및 '확인되지 않은 앱' 경고창이 뜹니다. 당황하지 마시고 **[고급]** 텍스트를 누른 뒤 **[안전하지 않음으로 이동]**을 클릭하여 '허용' 권한을 주시면 끝납니다.

### 배포/테스트 관련 안내
구글 앱스 스크립트(GAS)는 외부 인터넷에 떠 있기 때문에 `localhost` (로컬 개발 환경)로는 쏠 수 없습니다.
개발 환경에서 테스트를 해보시려면 `ngrok` 앱을 사용하여 로컬 포트포워딩을 하거나, 현재 코드를 실제 개발 서버(Vercel 등)에 커밋/배포한 뒤 배포 주소로 세팅해주셔야 실제 데이터가 DB에 박히는 것을 확인하실 수 있습니다.
