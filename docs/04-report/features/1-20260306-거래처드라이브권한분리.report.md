# O4-Report: 거래처별 구글 드라이브 권한(RBAC/ABAC) 매핑 및 UI 분리 작업 완료

## 작성일: 2026-03-06
## 1. 개요
createAI_v1의 '병원 관리자 승인 대기' 로직과 동일 한 구조로 구글 웹하드의 접근 권한을 거래처별로 원천 격리하는 다중 테넌트 기능을 구현 완료했습니다. 관리자는 거래처별 구역을 나누고 회원을 통제할 수 있습니다.

## 2. 작업 내역 및 결과
### 1) DB 스키마 분리 및 마이그레이션 (`server/src/db/schema.ts`)
- `HOSPITAL_ADMIN` 역할(`userRoleEnum`) 추가.
- 구글 드라이브 폴더 ID를 보유하는 `clients` 테이블 신설 빛 `users.clientId` 외래키 배정.
- Database Push (`drizzle-kit push`) 정상 적용 완료.

### 2) 백엔드 자동화 및 접근제어 (`server/src/routes/*.ts`)
- `/api/clients` (POST): 관리자가 신규 거래처명을 입력하면 즉시 구글 드라이브 `공유 자료실` 하위에 동일한 이름의 폴더를 API를 통해 직접 생성하고 DB에 저장(맵핑)합니다.
- `/api/admin/users/:id` (PATCH): 사용자 권한 수정 시 소속 병원(`clientId`) 선택이 가능하도록 업데이트.
- `/api/drive/folders/:folderId` & `/search`: 
  - 일반 가입증(`USER`등급)의 접근을 **HTTP 403 Forbidden**로 거절.
  - `HOSPITAL_ADMIN` 접속 시 최상위 부모 폴더 요청(`1SI_...`)이 본인의 `client.driveFolderId`로 강제 대체되어 해당 폴더 내부만 조회 가능하도록 격리.
  - 드라이브 하위 검색 시, `and '${clientFolderId}' in parents` 제약조건을 주입해 본인 병원 관련 파일 밖에는 열람 불가능하도록 보안 보완.

### 3) 프론트엔드 UI 수정 (`AdminPage.tsx`, `DrivePage.tsx`)
- **관리자 툴 (AdminPage.tsx)**: 거래처 간편 등록 폼 추가 및 역할 변경 콤보박스 선택 시 `HOSPITAL_ADMIN`일 때 거래처 목록(Select)이 토글로 등장하도록 동적인 액션을 구현.
- **자료실 거절 UI (DrivePage.tsx)**: 승인 대기 또는 깡통 사용자(`USER`) 로그인 시 UI를 통해 친절하고 명확한 예외 안내 메시지 및 아이콘(Error Empty State) 노출.

## 3. 테스트(Check) 방향 안내
관리자 계정으로 신규 접속하여, **관리자 메뉴에서 거래처(병원)를 1건 추가** 해 보시고 (실제 구글 드라이브에 생성되는지 확인), `TEST_HOSPITAL_...` 계정을 `HOSPITAL_ADMIN`와 해당 신규 병원으로 권한을 설정한 뒤 해당 계정으로 구글 드라이브 페이지가 어떻게 격리되는지 갭 체킹을 부탁드립니다.
