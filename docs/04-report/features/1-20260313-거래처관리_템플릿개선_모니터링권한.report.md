## 2. 계약종료 거래처 완전삭제 500 오류 수정

### 배경
계약종료 탭에서 `🗑️ 거래처 완전 삭제` 클릭 시 500 Internal Server Error 발생.

### 원인
- `users.clientId → clients.id` FK 제약 (CASCADE 없음)
- `tasks.clientId → clients.id` FK 제약 (CASCADE 없음)
- `db.delete(clients)` 실행 전 참조 레코드 미처리

### 해결
`DELETE /api/clients/:id` 엔드포인트에서 `db.delete(clients)` 실행 전:
1. `users.clientId` → `NULL` 업데이트
2. `tasks.clientId` → `NULL` 업데이트
3. 그 후 거래처 삭제 (clientServiceContracts는 CASCADE 처리됨)

### 수정 파일
| 파일 | 변경 내용 |
|------|-----------|
| `server/src/routes/client.ts` | import에 `users, tasks` 추가. DELETE 전 FK NULL 처리 로직 추가 |

> ⚠️ **중요**: DB 스키마의 FK 제약에 CASCADE가 걸려있지 않은 테이블이 있으므로, 추후 거래처 관련 삭제 로직 추가 시 동일한 패턴 적용 필요.

## 3. 업무 템플릿 날짜 질문 타입 추가

### 배경
템플릿 빌더에 날짜 형식 질문이 없었음.

### 변경 내용
두 가지 날짜 타입 추가:

| 타입 | 설명 | 응답 UI |
|------|------|---------|
| `date` (단일날짜) | 날짜 하나 | `<input type="date">` 1개 |
| `date_range` (기간날짜) | 시작일 ~ 종료일 | `<input type="date">` 2개 나란히 |

- 하단 버튼바: `📅 단일날짜` / `🗓️ 기간날짜` 버튼 추가
- 드롭다운: `📅 날짜 (단일)` / `📅 날짜 (기간: 시작일~종료일)` 옵션 추가
- `date_range` 응답값 형식: `{ start: "YYYY-MM-DD", end: "YYYY-MM-DD" }`

### 수정 파일
| 파일 | 변경 내용 |
|------|-----------|
| `client/src/pages/TemplatesPage.tsx` | QuestionType 타입 확장, 드롭다운·버튼바 추가 |
| `client/src/pages/TaskResponsePage.tsx` | `case 'date'`, `case 'date_range'` 렌더링 추가 |

## 4. 모니터링 HOSPITAL_ADMIN 권한 UI 분리

### 배경
병원관리자(HOSPITAL_ADMIN)에게 삭제 체크박스, 자동실행 중단 버튼이 노출되어 있었음.

### 변경 내용
`canDelete = user.role === 'ADMIN' || user.role === 'MANAGER'` 변수로 권한 구분:

| UI 요소 | ADMIN/MANAGER | HOSPITAL_ADMIN/USER |
|---------|--------------|---------------------|
| 결과 목록 전체선택 체크박스 | ✅ 표시 | ❌ 숨김 |
| 결과 항목別 체크박스 | ✅ 표시 | ❌ 숨김 |
| 자동실행중 버튼 (중단 가능) | ✅ 클릭 가능 | 🔒 클릭 불가 뱃지 표시 |
| 실행 버튼 | ✅ 표시 | ✅ 표시 (유지) |

### 수정 파일
| 파일 | 변경 내용 |
|------|-----------|
| `client/src/pages/MonitoringPage.tsx` | `canDelete` 변수 추가, 체크박스·자동실행 버튼 조건부 렌더링 |

## 5. 모니터링 템플릿 목록 거래처명 표시 개선

### 배경
- 최고관리자: `거래처: 그레이스병원` 정상 표시
- 병원관리자: `거래처: 알 수 없음` (또는 숫자 ID) 표시됨

### 원인
프론트엔드가 `/api/clients` 응답 배열로 클라이언트명을 조회했는데, HOSPITAL_ADMIN은 자신의 거래처만 반환받아서 다른 거래처 이름 조회 불가.

### 해결
서버의 `MonitoringService.getTemplates()`에서 `clients` 테이블을 **LEFT JOIN**하여 API 응답에 `clientName` 필드를 포함시킴. 프론트에서는 `t.clientName`을 직접 사용.

```
범위: googleplace | 거래처: 그레이스병원 | 수집: 10건 | 자동: 0 9 * * *
```

### 수정 파일
| 파일 | 변경 내용 |
|------|-----------|
| `server/src/services/monitoring/monitoringService.ts` | `getTemplates`에 `clients` LEFT JOIN 추가, `clientName` 필드 포함 |
| `client/src/pages/MonitoringPage.tsx` | `clients.find()` lookup → `t.clientName` 직접 사용 |
